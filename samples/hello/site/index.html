This page was generated using LambdaPad. It's Elixir! It's Simple!

LambdaPad lets you generate a site by melding data and templates. You can store data in TOML and write pages and posts using Markdown.

LambdaPad defines the entire site definition in a single lambdapad.exs file. Here's the file that was used to generate this page:

import Lambdapad

blog do
  pages "index" do
    set from: "pages/hello.md"
    set template: "index.html"
    set uri: "/"
    set var_name: "page"
  end
end

Let's break it down. Every LambdaPad file is an Elixir module importing Lambdapad. That's what this line says:

import Lambdapad

Next, the blog and pages block:

blog do
  pages "index" do
    set from: "pages/hello.md"
    set template: "index.html"
    set uri: "/"
    set var_name: "page"
  end
end

This returns an Elixir term that's used when defining the site and generating content from templates. For example, this snippet can be used in a template to generate a header containing the value of msg:

<h1>{{ page.msg }}</h1>

This snippet will render as:

<h1>Hello from Markdown!</h1>

That's it! When you run lpad, the site is generated!

Mardown Files

Everyone loves Markdown! So LambdaPad supports Markdown!

Here's hello.md:

msg: Hello from Markdown!

Markdown an easy way to maintain structured text that can be rendered in
various ways.

- It's human readable
- It's got a pretty easy syntax
- It can be rendered as HTML

Delightful!

LambdaPad supports a special header section for markdown files, which can contain metadata for the file. In this example we have a msg attribute, which we'll render thusly:

Hello from Markdown!

This is the code we used:

{{ page.msg }}

We can render the body of the markdown using the markdown_to_html filter. Here's the body as HTML:

msg: Hello from Markdown!

Markdown an easy way to maintain structured text that can be rendered in various ways.

  • It's human readable
  • It's got a pretty easy syntax
  • It can be rendered as HTML

Delightful!

Here's the code we used:

{{ 'pages/hello.md'|read_file|markdown_to_html }}
